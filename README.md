## 🌟 What is StudentHub?

> **StudentHub** is a full-stack Progressive Web App built for students. One platform to share study materials, discover internships and scholarships, collaborate on projects, attend campus events, and stay connected with your entire student community — all wrapped in a beautiful, fast, mobile-first experience.

## 🚀 Getting Started

### Prerequisites

- **Node.js** ≥ 18.x
- **npm** or **bun**
- A [Supabase](https://supabase.com) project

### Installation

```bash
# 1️⃣  Clone the repository
git clone <YOUR_GIT_URL>

# 2️⃣  Navigate into the project
cd share2all

# 3️⃣  Install dependencies
npm install

# 4️⃣  Set up environment variables
cp .env.example .env
```

### Environment Variables

Open `.env` and fill in your Supabase project credentials:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

```bash
# 5️⃣  Start the development server
npm run dev
```

> 🎉 App running at **http://localhost:8080**

## 🗄️ Database Schema

```mermaid
erDiagram
    profiles ||--o{ resources : uploads
    profiles ||--o{ projects : creates
    profiles ||--o{ opportunities : posts
    profiles ||--o{ events : organizes
    profiles ||--o{ shares : creates
    profiles ||--o{ user_roles : has

    projects ||--o{ project_members : has
    events ||--o{ event_rsvps : has
    conversations ||--o{ conversation_members : has
    conversations ||--o{ messages : contains
    notifications ||--o{ notification_reads : tracks

    profiles {
        uuid id PK
        uuid user_id FK
        string avatar_url
        string bio
        string department
        string year_of_study
        string[] skills
        string github_url
        string linkedin_url
    }
    user_roles {
        uuid id PK
        uuid user_id FK
        enum role "admin|moderator|user"
    }
    resources {
        uuid id PK
        string title
        string subject
        string category
        string file_path
        string course_code
    }
    projects {
        uuid id PK
        string title
        string description
        string status
        string[] tags
    }
    events {
        uuid id PK
        string title
        timestamp event_date
        string category
        string location
    }
    opportunities {
        uuid id PK
        string title
        string organization
        string category
        timestamp deadline
    }
    shares {
        uuid id PK
        string share_token
        string share_type
        string access_method
        string access_code
        int view_count
    }
```

## 🔒 Security

<div align="center">

| ✅ | Security Measure |
|----|-----------------|
| 🔐 | Row-Level Security (RLS) on every database table |
| 🛡️ | Role-based access control — Admin, Moderator, User |
| ⚙️ | Server-side role validation via `has_role()` security definer |
| 📁 | Authenticated-only access to protected file storage |
| 📧 | Email verification required on signup |
| 🚧 | Client-side protected routes with auth guards |
| 🔑 | No sensitive credentials exposed in client code |

## ☁️ Deployment

The app is configured for **Vercel** deployment. To deploy:

1. Push to your connected Git repository
2. Vercel auto-detects the Vite config and deploys

Add your environment variables in the Vercel project settings:

```
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```
